## 核心参数速查

### 生成参数
| 参数 | 选项 | 说明 |
|------|------|------|
| 分辨率 | 480p (1积分) / 1080p (5积分) | 1080p 满足平台需求 |
| 模式 | 文生视频 / 图生视频 / 多镜头叙事 | 多镜头效率最高 |
| 单条时长 | 1-15s | 5s = 1080p最佳性价比 |
| 上传上限 | 9图 + 3视频 + 3音频 | 合计12文件 |

### @ 引用语法速查
```
@Image1  作为主角色参考
@Image2  作为配角参考
@Image3  作为场景参考/背景环境
@Image4  作为道具/武器参考
@Image5  作为服装风格参考
@Image6  作为色彩方案参考
@Video1  作为镜头运动参考
@Video2  作为动作节奏参考
@Audio1  作为背景音乐
@Audio2  作为角色声音/对话节奏参考
```

### 固定负面提示词
```
模糊、低画质、面部变形、多余手指、水印、文字、字幕、logo、ui界面
```

## 模式选择指南

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 单镜头精调 | 图生视频 + 垫图 | 精细控制 |
| 叙事短剧 | 多镜头叙事 | 一次生成，保持一致 |
| 漫剧段落 | 多镜头叙事 + 九宫格垫图 | 整段生成 |
| 产品展示 | 图生视频（逐条） | 每条精细调整 |
| 实验影像 | 首尾帧 或 全能参考 | 发挥AI创意 |

## 提示词最佳实践

### ✅ 推荐做法
```
- 用现在进行时描述动作："旋转着""缓慢走向""突然爆发"
- 镜头运动用自然语言："固定机位""缓慢推进""镜头环绕"
- 一个镜头只描述一种运镜
- 台词控制在15字以内
- 包含画质关键词：8K超清/电影级画质/原生实拍质感
- 每 Cut 都 @ 引用同一张角色参考图
```

### ❌ 避免做法
```
- "首帧...尾帧..."（旧版语法，Seedance 2.0 不支持）
- "先推后拉再摇"（多种运镜混用）
- 抽象描述："他感到悲伤"→ 应写"他单手掩面，肩膀微颤"
- 超过15s不切分
- 依赖文本叠加特效
- 多角色同时说话（口型同步只能跟一个）
- 负面提示词超过10个关键词
```

## 成本优化策略

| 阶段 | 方案 | 节省 |
|------|------|------|
| 试错阶段 | 480p 测试 (1积分) | 省80% |
| 确认后 | 1080p 生成 (5积分) | — |
| 多版本 | 同提示词不同seed值，保留最优 | — |
| 失败处理 | 修改提示词而非重新随机生成 | 减少无效消耗 |

## 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 角色脸变了 | 没垫图/@引用不一致 | 上传三视图，所有Cut统一 @Image1 |
| 口型对不上 | 台词太长/多角色说话 | ≤15字，一个Cut一个角色说话 |
| 画面抖动 | 多运镜混用 | 一个镜头一个运镜方式 |
| 画面模糊 | 选了480p/没写画质词 | 1080p + "8K超清锐利细节" |
| 光跳变 | 各Cut光影描述不一致 | 锁定光影方案，逐Cut复制 |
| 动作不自然 | 描述太抽象 | 用具体的物理动作描述 |
| 生成慢 | 高峰期排队 | 错峰使用，或购买Pro去广告排队 |
